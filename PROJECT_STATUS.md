# PROJECT_STATUS — USDT-USDC GUI

## 1. Что это за проект
- **USDT-USDC** — GUI-only desktop-приложение на **PySide6** для мониторинга котировок пары USDT/USDC (без API, без бизнес-логики, только мок-данные).
- Текущий этап: **только GUI** (mock data, без подключения бирж).

## 2. Текущая структура проекта
Ключевые папки и файлы:
- `src/app/main.py` — **точка входа**.
- `src/gui/` — основной GUI (окно, виджеты, layout).
- `src/core/mock_data.py` — генерация мок-данных для таблиц.
- `requirements/` — расширенный список зависимостей (включает PySide6).
- `requirements.txt` — основной файл зависимостей (сейчас включает `-r requirements/base.txt`).

## 3. Как запускается программа
Команда запуска из корня репозитория:
```bash
python -m src.app.main
```
Окружение: активированное виртуальное окружение `.venv` с установленными зависимостями (включая **PySide6** и **loguru**).

## 4. Текущий статус запуска
⚠️ **Запуск остановлен на импорте PySide6 в текущем окружении.**
- Фактическая попытка запуска из корня репозитория:
  ```bash
  python -m src.app.main
  ```
  завершилась с ошибкой `ModuleNotFoundError: No module named 'PySide6'`.
- GUI создаётся в `src/app/main.py` через `QApplication` и `MainWindow`, с показом окна via `window.show()`.
- Из-за отсутствия PySide6 в среде запуска фактическое отображение окна не подтверждено.

## 5. Найденные проблемы
- **В текущем окружении отсутствует PySide6.**
  - При запуске `python -m src.app.main` возникает `ModuleNotFoundError: No module named 'PySide6'`.
  - Для запуска требуется установить зависимости из `requirements.txt` (включает `requirements/base.txt` с `PySide6`).

## 7. Что нужно сделать дальше
- Установить зависимости в виртуальном окружении:
  ```bash
  pip install -r requirements.txt
  ```
- Запустить приложение:
  ```bash
  python -m src.app.main
  ```
- Если окно не открывается, проверить:
  - что `PySide6` установлена;
  - что запуск происходит из корня репозитория;
  - что используется правильная версия Python из `.venv`.
