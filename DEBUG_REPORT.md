# DEBUG_REPORT — USDT-USDC GUI

## 1. Фактическая структура проекта
- `src/app/main.py` — точка входа (entrypoint).
- `src/gui/main_window.py` — главное окно GUI.
- `src/gui/widgets/` — набор виджетов интерфейса.
- `src/core/mock_data.py` — генерация мок-данных.

## 2. Команда запуска
Из корня репозитория:
```bash
python -m src.app.main
```

## 3. Причина исходной ошибки
- В этом окружении запуск падает на импорте `PySide6`:
  - `ModuleNotFoundError: No module named 'PySide6'`.
- Это означает, что зависимости GUI не установлены в активном интерпретаторе.

## 4. Что исправлено
- Кодовая структура entrypoint уже соответствует команде запуска (`src/app/main.py`).
- Импорты GUI абсолютные и начинаются с `src.` — конфликтов пакетов не обнаружено.

## 5. Текущий статус
- **GUI не запускается в этом окружении**, пока не установлены зависимости.
- После установки зависимостей через:
  ```bash
  pip install -r requirements.txt
  ```
  команда `python -m src.app.main` должна запускать GUI.
